## Security

- [x] [sec-1|high] Report temp-file template is non-portable and non-random (plugins/my-skills/skills/explain-codebase/SKILL.md:384)
  fingerprint: security|plugins/my-skills/skills/explain-codebase/SKILL.md|report-temp-file-template-is-non-portable-and-non-random
  Rationale: The sec-1 fix calls mktemp with mid-template Xs (.explain-XXXXXXXX.html.tmp); BSD/macOS mktemp leaves the Xs literal — creating the fully predictable constant path every run (verified by execution) — and GNU mktemp errors out entirely, so on Linux the report write always fails; only the O_EXCL create remains, and pre-creating the constant name is a persistent denial.
  Fix: Move the Xs to the end of the template (mktemp "$outreal/.explain-html-XXXXXXXX"), rename to the .html name only at the final mv, and assert the created basename differs from the template so a literal-X result aborts.
