Proposed Commit Message

feat(export): add async, resumable export pipeline

Decouples large exports from the request path: a request returns a
tracking ID, a background worker writes the artifact to object store,
and downloads support HTTP range requests so they resume rather than
restart. Adds expiry, idempotency, and completion notifications.

- Add asynchronous export request endpoint that returns a tracking ID
- Background worker produces a resumable artifact with expiry + idempotency

## Test plan

- [ ] Interrupt a download mid-transfer and confirm it resumes via range request
- [ ] Re-request an identical export and confirm the existing artifact is reused
